## 软件工程基础

### 软件生命周期模型
**瀑布模型**
- 特点：线性顺序，阶段清晰
- 适用：需求明确、稳定的项目
- 缺点：缺乏灵活性，风险高

**螺旋模型**
- 特点：风险驱动，迭代开发
- 四个象限：制定计划→风险分析→实施工程→客户评估
- 适用：大型复杂项目

**敏捷模型**
- 核心价值：个体>流程、软件>文档、合作>合同、响应>计划
- 常见方法：Scrum、XP、看板

### 软件开发方法
**结构化方法**
- 分析：数据流图(DFD)、数据字典、加工说明
- 设计：模块结构图、详细设计
- 原则：模块化、抽象、信息隐藏

**面向对象方法**
- 分析：用例图、类图、时序图
- 设计：设计模式、SOLID原则
- 实现：封装、继承、多态

### 软件质量模型
**ISO 9126质量模型**
1. 功能性(Functionality)
2. 可靠性(Reliability)
3. 易用性(Usability)
4. 效率(Efficiency)
5. 可维护性(Maintainability)
6. 可移植性(Portability)

## 面向对象技术

### 基本概念
**类与对象**
- 类：对象的模板和蓝图
- 对象：类的实例
- 属性：对象的状态
- 方法：对象的行为

**三大特性**
1. **封装**：数据和方法的结合，信息隐藏
2. **继承**：子类获得父类的属性和方法
3. **多态**：同一接口的不同实现

### UML图形
**用例图(Use Case Diagram)**
- 参与者(Actor)：系统外部实体
- 用例(Use Case)：系统功能
- 关系：关联、包含、扩展、泛化

**类图(Class Diagram)**
- 类的表示：类名、属性、方法
- 关系：关联、聚合、组合、依赖、继承

**时序图(Sequence Diagram)**
- 对象生命线
- 消息传递
- 激活框

**状态图(State Diagram)**
- 状态、转换、事件、动作
- 初始状态、终止状态

### 设计模式
**创建型模式**
- 单例模式：确保类只有一个实例
- 工厂模式：创建对象的接口
- 建造者模式：复杂对象的构建

**结构型模式**
- 适配器模式：接口转换
- 装饰器模式：动态添加功能
- 外观模式：简化接口

**行为型模式**
- 观察者模式：一对多依赖关系
- 策略模式：算法族的封装
- 命令模式：请求封装为对象

## 数据库技术

### 关系数据库理论
**关系模型**
- 关系：二维表
- 元组：行(记录)
- 属性：列(字段)
- 域：属性的取值范围

**关系代数**
- 选择(σ)：按条件选择元组
- 投影(π)：选择属性
- 连接(⋈)：表的连接操作
- 并(∪)、交(∩)、差(-)

### 数据库设计
**ER模型**
- 实体(Entity)：矩形表示
- 属性(Attribute)：椭圆表示
- 联系(Relationship)：菱形表示
- 基数：1:1、1:N、M:N

**规范化理论**
**第一范式(1NF)**：属性不可分割
**第二范式(2NF)**：消除部分函数依赖
**第三范式(3NF)**：消除传递函数依赖
**BCNF**：消除主属性对候选键的部分和传递依赖

### SQL语言
**数据定义(DDL)**
```sql
CREATE TABLE, ALTER TABLE, DROP TABLE
CREATE INDEX, DROP INDEX
```

**数据操作(DML)**
```sql
SELECT, INSERT, UPDATE, DELETE
```

**数据控制(DCL)**
```sql
GRANT, REVOKE
COMMIT, ROLLBACK
```

**常用查询**
- 连接查询：内连接、外连接
- 子查询：相关子查询、非相关子查询
- 聚合函数：COUNT、SUM、AVG、MAX、MIN
- 分组：GROUP BY、HAVING

## 数据结构与算法

### 基本数据结构
**线性结构**
- 数组：随机访问，插入删除效率低
- 链表：动态分配，插入删除效率高
- 栈：LIFO，后进先出
- 队列：FIFO，先进先出

**树结构**
- 二叉树：每个节点最多两个子节点
- 二叉搜索树：左子树<根<右子树
- 平衡二叉树：任意节点左右子树高度差≤1
- B树：多路搜索树，适合外存

**图结构**
- 有向图、无向图
- 邻接矩阵、邻接表
- 深度优先搜索(DFS)
- 广度优先搜索(BFS)

### 算法分析
**时间复杂度**
- O(1)：常数时间
- O(log n)：对数时间
- O(n)：线性时间
- O(n log n)：线性对数时间
- O(n²)：平方时间

**空间复杂度**
- 算法执行过程中所需的存储空间

### 经典算法
**排序算法**
- 冒泡排序：O(n²)
- 快速排序：平均O(n log n)
- 归并排序：O(n log n)
- 堆排序：O(n log n)

**查找算法**
- 顺序查找：O(n)
- 二分查找：O(log n)
- 哈希查找：平均O(1)

## 计算机系统基础

### 计算机硬件
**CPU**
- 运算器：算术逻辑单元(ALU)
- 控制器：指令译码、控制信号
- 寄存器：程序计数器(PC)、指令寄存器(IR)

**存储器层次**
- 寄存器：速度最快，容量最小
- 高速缓存(Cache)：SRAM
- 主存储器：DRAM
- 辅助存储器：硬盘、SSD

**I/O系统**
- 程序控制I/O
- 中断驱动I/O
- DMA方式

### 操作系统
**进程管理**
- 进程状态：就绪、运行、阻塞
- 进程调度：FCFS、SJF、时间片轮转
- 进程同步：信号量、管程、消息传递

**内存管理**
- 分区管理：固定分区、动态分区
- 分页管理：页表、TLB
- 分段管理：段表
- 虚拟内存：请求分页、页面置换算法

**文件系统**
- 文件组织：顺序、索引、散列
- 目录结构：单级、两级、树形
- 磁盘调度：FCFS、SSTF、SCAN

## 网络与安全

### 网络协议
**OSI七层模型**
1. 物理层：比特流传输
2. 数据链路层：帧传输、差错控制
3. 网络层：路由选择、IP协议
4. 传输层：端到端通信、TCP/UDP
5. 会话层：会话管理
6. 表示层：数据格式转换
7. 应用层：网络服务

**TCP/IP协议族**
- IP协议：无连接、不可靠
- TCP协议：面向连接、可靠传输
- UDP协议：无连接、不可靠、高效
- HTTP协议：超文本传输协议

### 网络安全
**安全威胁**
- 被动攻击：窃听、流量分析
- 主动攻击：伪装、重放、修改、拒绝服务

**安全机制**
- 认证：身份验证
- 授权：访问控制
- 加密：对称加密、非对称加密
- 数字签名：身份认证、数据完整性

**加密技术**
- 对称加密：DES、AES
- 非对称加密：RSA、ECC
- 散列函数：MD5、SHA

## 软件测试

### 测试基础
**测试目的**
- 发现软件缺陷
- 验证软件功能
- 评估软件质量

**测试原则**
- 测试显示缺陷的存在
- 穷尽测试不可能
- 测试活动应尽早开始
- 缺陷集群性

### 测试方法
**按测试技术分类**
- 黑盒测试：等价类划分、边界值分析、判定表
- 白盒测试：语句覆盖、判定覆盖、条件覆盖、路径覆盖
- 灰盒测试：结合黑盒和白盒

**按测试阶段分类**
- 单元测试：模块级测试
- 集成测试：模块间接口测试
- 系统测试：整个系统测试
- 验收测试：用户验收

### 测试用例设计
**等价类划分**
- 有效等价类：满足需求的输入
- 无效等价类：不满足需求的输入

**边界值分析**
- 测试边界值及其邻近值
- 通常选择边界值±1

**判定表**
- 条件桩、动作桩
- 条件项、动作项
- 适用于复杂逻辑关系

## 项目管理

### 项目管理基础
**项目特征**
- 临时性：有明确的开始和结束时间
- 独特性：产品或服务的独特性
- 渐进明细：逐步完善项目细节

**项目管理过程组**
1. 启动过程组
2. 规划过程组
3. 执行过程组
4. 监控过程组
5. 收尾过程组

### 项目管理知识域
**范围管理**
- 需求收集
- 范围定义
- 工作分解结构(WBS)
- 范围控制

**时间管理**
- 活动定义
- 活动排序
- 活动资源估算
- 活动历时估算
- 制定进度计划
- 控制进度

**成本管理**
- 成本估算
- 成本预算
- 成本控制

**质量管理**
- 质量规划
- 质量保证
- 质量控制

### 项目计划技术
**网络图**
- 活动节点图(AON)
- 活动箭线图(AOA)
- 关键路径法(CPM)
- 计划评审技术(PERT)

**甘特图**
- 横道图表示
- 直观显示进度
- 便于跟踪控制

3. **UML图设计**：用例图、类图、时序图，分值10-15分
4. **面向对象设计**：设计模式应用，分值10-15分
5. **算法设计**：伪代码或流程图，分值10-15分

- 多练习下午题的标准答题格式

- 掌握数据库设计的基本步骤
- 了解主要设计模式的应用场景

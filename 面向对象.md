# 面向对象基础概念

本文对面向对象编程（OOP）的基本概念进行梳理，包含对象、属性、方法、封装、访问修饰符及类的基础知识，并配合实例、表格和流程步骤，方便理解与复习。

---

## 一、对象（Object）

**概念：**  
- 英文单词：`Object`  
- 含义：最原始的意思是物体，可看得见、摸得着的事物  
- 面向对象思想：**万事万物都是对象**  

**特点：**  
1. **对象名**：每个对象有唯一名字（如猫大、猫二）  
2. **静态特征（属性/状态/数据/变量）**  
   - 静态特征是对象的固定特性  
   - 示例：
        - 毛色：蓝色  
        - 高度：120  
3. **动态特征（方法/函数/行为/功能）**  
   - 动态特征是对象可以执行的动作或行为  
   - 示例：
        - `喵喵叫()`  
        - `跑()`  

**总结表：**

| 特征类型 | 专业术语 | 示例 |
|----------|----------|------|
| 静态特征 | 属性 / 状态 / 数据 / 变量 | 毛色、身高 |
| 动态特征 | 方法 / 函数 / 功能 | 喵喵叫、跑 |

---

## 二、封装（Encapsulation）

**概念：**  
- 将对象的属性和方法绑定在一起，并对外界访问加以限制  
- 目的是保护对象内部数据，防止被随意修改  

**访问修饰符（控制访问权限）：**

| 符号 | 名称 | 访问范围 |
|------|------|----------|
| `+` | public（公共） | 内部、外部、子类、同包都可访问 |
| `-` | private（私有） | 仅类内部可访问 |
| `#` | protected（受保护） | 同胞类及子类可访问 |
| （空） | default（默认） | 同胞类可访问，外部不可访问 |

**消息传递：**  
- 外部向对象传递数据或指令称为消息传递  
- 消息属于外界的数据，不是对象内部的状态  
- 接收消息需要**接口**（Interface）  

---

## 三、对象的组成部分

1. **标识（Object ID）**：对象名或ID，唯一标识对象  
2. **属性（静态特征）**：对象的状态，可能不同对象值不同，但名称相同  
3. **方法（动态特征）**：对象的行为，不同对象同类方法名称相同，但执行内容可不同

## 四、类（Class）

**概念：**  
- 类是对象的抽象模板  
- 定义了一组相似对象的**共有特征**和**行为**  
- 对象由类实例化生成  

**类与对象关系：**

| 对象 | 类 |
|------|----|
| 独立实例 | 模板 |
| 具有对象名、属性、方法 | 定义属性名称、方法名称 |
| 属性值可不同 | 属性名称一致，方法实现可统一 |

**示例：猫类与猫对象**

猫类
├─ 属性：毛色、身高
└─ 方法：喵喵叫、跑()

猫大（对象）
├─ 毛色：蓝色
└─ 身高：120

猫二（对象）
├─ 毛色：白色
└─ 身高：100

- **共享**：同一类的对象共享属性名、方法名  
- **不同**：对象名、属性值可能不同

## 五、面向对象关键知识点

1. **封装**  
   - 属性和方法绑定到对象内部  
   - 外界通过接口访问  
2. **抽象**  
   - 只关注对象的**必要特征和行为**  
3. **继承**  
   - 子类继承父类属性和方法  
4. **多态**  
   - 不同对象可以以相同方法名执行不同操作  

**访问控制示例：**

类内部：
public属性和方法：内部外部均可访问
private属性和方法：仅内部访问
protected属性和方法：内部及子类访问
default属性和方法：同包访问

## 六、总结流程

1. 识别对象 → 名称、属性、方法  
2. 理解属性 → 静态特征、值可不同  
3. 理解方法 → 动态特征、功能实现  
4. 封装 → 将属性和方法绑定，并限制访问  
5. 访问修饰符 → 控制外界访问权限  
6. 类 → 抽象对象模板，共享特征与行为  
7. 面向对象四大特性 → 封装、抽象、继承、多态

# 面向对象基础与开发流程

## 一、对象与类

### 1. 对象（Object）
- **定义**：现实世界中具体存在的事物称为对象。
- **特征**：
  - 静态特征：属性（Properties）
  - 动态特征：方法/行为（Methods）
- **封装**：将对象的属性和方法放在一起，外部通过接口访问。
  
**示例：**
    class Cat:
        def __init__(self, name, color):
            self.name = name
            self.color = color
        def meow(self):
            print("喵喵")

### 2. 类（Class）
- **定义**：对象的抽象模板，描述对象的共性。
- **数量差异**：
  - 类：一个
  - 对象：多个实例化对象
- **抽象 vs 具体**：
  - 类是抽象概念
  - 对象是具体实例
- **别名**：
  - 数据成员：静态特征属性
  - 成员函数：动态特征行为

### 3. 类的分类

| 类别 | 作用 | 特征 | 示例 |
|------|------|------|------|
| 实体类（数据类） | 存储大量数据 | 对应现实存在的物体 | 用户、账户、学生信息 |
| 功能类（接口类/边界类） | 提供功能和接口 | 仅方法为主 | 二维码支付功能 |
| 控制类 | 控制业务逻辑 | 调用实体类和功能类方法实现业务 | 支付控制器 |

**真题示例**：
- 客户 → 实体类  
- 二维码 → 功能类（接口类/边界类）

---

## 二、面向对象三大特征

### 1. 封装（Encapsulation）
- 将属性与方法放入对象，外部通过接口访问
- 封装的好处：减少重复代码，提高安全性和可维护性

### 2. 继承（Inheritance）
- **概念**：子类共享父类已有属性和方法
- **术语**：
  - 父类：超类、泛型类
  - 子类：继承父类，增加特有属性/方法
- **特点**：
  - 共享（不用重复写父类代码）
  - 可新增特有属性与方法
  - 可重写父类方法
- **语法**：
    class Cat(Animal):
        pass

- **Java 继承限制**：不支持多继承（R异性），避免冲突  
- **C++**：支持多继承

### 3. 多态（Polymorphism）
- **定义**：同样的方法调用不同对象产生不同结果
- **类型**：
  - **过载多态（方法重载）**：同一类中方法名相同，参数不同
  - **重写多态（方法重写/覆盖）**：子类重写父类方法
  - **参数多态**：父类类型参数接收子类对象
  - **包含多态**：集合中使用父类类型存储子类对象
  - **强制多态**：类型强制转换

**示例**：
    class Animal:
        def call(self):
            pass
    
    class Cat(Animal):
        def call(self):
            print("喵喵")
    
    class Dog(Animal):
        def call(self):
            print("汪汪")
    
    # 动态绑定实现多态
    a = Cat()
    b = Dog()
    animals = [a, b]
    for animal in animals:
        animal.call()  # 输出分别为：喵喵、汪汪

- **动态绑定**：运行时确定变量实际类型  
- **静态绑定**：编译时已确定变量类型

## 三、方法概念

| 方法类型 | 定义 | 发生位置 |
|----------|------|----------|
| 重载（Overload） | 同名方法参数不同 | 同一类 |
| 重写（Override/覆盖） | 子类重新定义父类方法 | 子父类之间 |

---

## 四、接口与抽象类

- **抽象类（abstract class）**：
  - 包含抽象方法，子类必须实现
  - 确保多态性实现
- **接口（interface）**：
  - 所有方法默认抽象
  - 提供统一方法定义，减少子类重复实现

**示例**：
    from abc import ABC, abstractmethod
    
    class Animal(ABC):
        @abstractmethod
        def call(self):
            pass

    class Dog(Animal):
        def call(self):
            print("汪汪")

## 五、面向对象开发流程

### 四大阶段
1. **分析阶段**
   - 认定对象
   - 组织对象
   - 对象之间相互作用
   - 基于对象操作
2. **设计阶段**
   - 识别类及对象
   - 定义属性与方法
3. **程序设计阶段**
   - 确定开发标准
   - 选择面向对象语言（Java、Python）
4. **测试阶段**
   - 从方法 → 类 → 模块 → 系统逐层测试

### 开发经验（七大原则）

| 原则 | 说明 |
|------|------|
| 单一职责 | 类职责单一，便于维护 |
| 开放封闭 | 扩展开放，修改封闭 |
| 里氏替换 | 子类对象可替换父类对象 |
| 依赖倒置 | 用接口/抽象类接子类对象 |
| 接口隔离 | 接口专一，避免臃肿 |
| 组合重用 | 优先组合，减少继承 |
| 迪米特原则 | 最少知识原则，降低耦合 |

**组合重用示例**：
- CPU：1万种  
- 内存：1万种  
- 组合方式：通过组合而非继承实现1亿种不同手机型号

## 六、总结

1. **对象**：具体存在，封装属性与方法  
2. **类**：抽象模板，分类：实体类、功能类、控制类  
3. **封装**：属性与方法结合，外部通过接口访问  
4. **继承**：共享父类代码，可新增/重写方法  
5. **多态**：同方法调用不同对象产生不同结果，基于动态绑定  
6. **接口/抽象类**：保证多态性，实现统一方法规范  
7. **开发流程**：分析 → 设计 → 程序设计 → 测试  
8. **设计原则**：七大经验提高程序可维护性与扩展性

# 信息安全基础知识笔记

信息安全的核心是保证信息在存储、传输和使用过程中的安全性，通常归纳为五个基本要素。下面详细整理学习要点、技术实现及生活中的例子，便于复习。

---

## 一、信息安全五大基本要素

| 要素 | 英文 | 定义 | 技术实现/例子 |
|------|------|------|----------------|
| 机密性 | Confidentiality | 确保信息不暴露给未授权的实体或进程 | 加密（对称加密、非对称加密）、访问控制 |
| 完整性 | Integrity | 信息只有在得到授权的人允许下才能修改，并能判断是否被篡改 | 摘要算法（如 MD5、SHA）、数字签名 |
| 可用性 | Availability | 得到授权的实体在需要时可访问数据，攻击者不能阻碍正常访问 | 高可用架构、抗 DDoS、防止资源耗尽 |
| 可控性 | Control / Manageability | 控制授权方位的信息流向及行为方式，防止信息泛滥 | 权限管理、访问策略、防止信息泄露 |
| 可审查性 | Auditability | 对信息的操作行为进行记录，以便事后分析、审计 | 日志记录、审计系统、事件追踪 |

---

## 二、五大要素详细说明

### 1. 机密性（Confidentiality）
- **含义**：信息只允许授权主体访问。
- **常用技术**：
  - 数据加密（AES、RSA 等）
  - 访问控制列表（ACL）
- **生活例子**：
  - 私人聊天信息只允许聊天对象查看
  - 银行账户信息不被外泄

### 2. 完整性（Integrity）
- **含义**：信息未经授权不能被修改，且可以检测篡改。
- **实现方式**：
  - 摘要算法（MD5、SHA-256）
  - 数字签名、校验和
- **生活例子**：
  - 下载软件包时验证哈希值
  - 邮件或文件内容被篡改会提示不一致

### 3. 可用性（Availability）
- **含义**：授权用户在需要时可以访问信息。
- **攻击防护**：
  - 拒绝服务攻击（DoS/DDoS）防护
  - 负载均衡与高可用系统
- **生活例子**：
  - 节假日抢票网站拥堵时，有恶意占用资源导致正常用户无法访问
  - 服务器宕机导致服务中断

### 4. 可控性（Control / Manageability）
- **含义**：授权方能够控制信息的流向及使用行为。
- **技术措施**：
  - 访问权限控制
  - 信息流向策略
  - 防止信息泛滥（如垃圾短信、垃圾邮件）
- **生活例子**：
  - 手机接收垃圾短信、垃圾电话的问题
  - 企业内部敏感信息只允许内部员工访问

### 5. 可审查性（Auditability）
- **含义**：对信息处理行为进行记录，以便事后分析与审查。
- **技术手段**：
  - 系统日志记录
  - 审计跟踪
- **生活例子**：
  - 银行交易记录可追溯
  - 系统安全事件发生后可分析原因

---

## 三、信息安全概念总结

1. **不可抵赖性（Non-repudiation）**
   - 定义：发送方无法否认自己发送的信息。
   - 实现方式：
     - 数字签名
     - 公钥基础设施（PKI）
   - 例子：
     - A 发邮件给 B，使用数字签名，A 无法否认邮件是自己发送的

2. **真题解析**
   - 题目：所有资源只能由授权方以授权方式进行修改。
   - 对应要素：**完整性**（未经授权不可修改信息）
   - 相关技术：摘要算法、访问控制、权限验证

---

## 四、五要素之间的关系

- **机密性 + 可控性** → 防止信息泄露与泛滥  
- **完整性 + 可审查性** → 确保信息可信，并可追踪篡改行为  
- **可用性** → 保证授权用户能够正常访问信息  
- **不可抵赖性** → 对信息操作行为进行认证与确认

---

## 五、生活中的安全例子总结

| 场景 | 对应要素 | 技术/措施 |
|------|----------|-----------|
| 抢票高峰网站无法访问 | 可用性 | 高可用架构、抗 DDoS |
| 手机收到垃圾短信 | 可控性 | 信息流向控制、权限管理 |
| 下载软件验证哈希 | 完整性 | 摘要算法、校验和 |
| 企业内部文件只允许特定人员访问 | 机密性 | 访问控制、加密 |
| 银行交易记录留存 | 可审查性 | 日志审计、事件追踪 |
| 邮件签名防抵赖 | 不可抵赖性 | 数字签名、公钥验证 |

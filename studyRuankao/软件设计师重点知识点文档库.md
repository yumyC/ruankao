## 数据结构与算法
**重要程度：高 | 考查频度：很高**

### 核心知识点

#### 1. 线性结构
- **栈（Stack）**：后进先出（LIFO）
  - 基本操作：push、pop、top
  - 应用：表达式求值、函数调用、括号匹配
- **队列（Queue）**：先进先出（FIFO）
  - 基本操作：enqueue、dequeue、front
  - 应用：广度优先搜索、任务调度

#### 2. 树与二叉树
- **二叉树遍历**：
  - 前序遍历：根→左→右
  - 中序遍历：左→根→右
  - 后序遍历：左→右→根
- **哈夫曼树**：最优二叉树，用于数据压缩

#### 3. 图
- **遍历算法**：
  - 深度优先搜索（DFS）
  - 广度优先搜索（BFS）
- **最小生成树**：Prim算法、Kruskal算法
- **拓扑排序**：有向无环图的线性排序

#### 4. 常用算法
- **排序算法**：冒泡、选择、插入、快速、归并、堆排序
- **查找算法**：顺序查找、二分查找、哈希查找
- **动态规划**：最优子结构、状态转移方程
- **回溯算法**：试探性搜索，回退机制

### 典型例题

**例题1：栈的应用**
```
表达式 "3 + 4 * 2 - 1" 的后缀表达式是？
A) 3 4 2 * + 1 -
B) 3 4 + 2 * 1 -
C) 3 4 2 + * 1 -
D) 3 4 2 * 1 + -

答案：A
解析：中缀转后缀，运算符优先级：* > + > -
```

**例题2：二叉树遍历**
```
已知二叉树的前序遍历为ABDECF，中序遍历为DBEACF，求后序遍历。
A) DEBFCA
B) DEBACF
C) DEAFCB
D) DEBFAC

答案：A
解析：根据前序和中序可以确定树的结构，然后进行后序遍历。
```

## 操作系统
**重要程度：高 | 考查频度：高**

### 核心知识点

#### 1. 进程管理
- **进程状态**：新建、就绪、运行、阻塞、终止
- **同步与互斥**：
  - PV操作（信号量）
  - 临界区、互斥锁
- **死锁**：
  - 必要条件：互斥、占有等待、不可抢占、循环等待
  - 预防、避免、检测、解除
- **调度算法**：
  - 先来先服务（FCFS）
  - 短作业优先（SJF）
  - 时间片轮转（RR）
  - 优先级调度

#### 2. 存储管理
- **分页存储**：
  - 页表、页面大小
  - 地址转换：逻辑地址→物理地址
- **分段存储**：段表、段内地址
- **虚拟存储器**：
  - 页面置换算法：FIFO、LRU、OPT
  - 工作集模型

#### 3. 文件与设备管理
- **文件结构**：
  - 逻辑结构：顺序、索引、链接
  - 物理结构：连续、链接、索引
- **磁盘调度算法**：
  - FCFS、SSTF、SCAN、C-SCAN

### 典型例题

**例题1：进程调度**
```
有3个进程P1、P2、P3，到达时间分别为0、1、2，服务时间分别为3、6、4。
采用短作业优先调度算法，P2的完成时间是？
A) 9
B) 10
C) 13
D) 7

答案：C
解析：按到达时间和服务时间计算，P1(0-3)→P3(3-7)→P2(7-13)
```

**例题2：页面置换**
```
采用LRU页面置换算法，内存块数为3，页面访问序列为1,2,3,4,1,2,5,1,2,3,4,5，
缺页次数是？
A) 9
B) 10
C) 11
D) 8

答案：A
解析：模拟LRU算法执行过程，统计缺页次数。
```

## 数据库系统
**重要程度：高 | 考查频度：高**

### 核心知识点

#### 1. 数据模型与E-R图
- **实体-联系模型**：
  - 实体、属性、联系
  - 一对一、一对多、多对多关系
- **E-R图转关系模式**：
  - 实体→关系
  - 联系→关系或外键

#### 2. 关系代数与规范化
- **基本运算**：
  - 选择（σ）、投影（π）、连接（⋈）
  - 并（∪）、差（-）、交（∩）
- **范式理论**：
  - 第一范式（1NF）：原子性
  - 第二范式（2NF）：消除部分函数依赖
  - 第三范式（3NF）：消除传递函数依赖
  - BCNF：消除主属性对候选键的部分和传递依赖

#### 3. SQL语言
- **数据定义**：CREATE、ALTER、DROP
- **数据查询**：SELECT、WHERE、GROUP BY、HAVING、ORDER BY
- **数据控制**：GRANT、REVOKE

#### 4. 事务与并发控制
- **ACID特性**：
  - 原子性（Atomicity）
  - 一致性（Consistency）
  - 隔离性（Isolation）
  - 持久性（Durability）
- **并发问题**：丢失修改、不可重复读、读脏数据
- **封锁协议**：一级、二级、三级封锁协议

### 典型例题

**例题1：关系代数**
```
关系R(A,B,C)和S(C,D,E)，R⋈S的结果包含哪些属性？
A) A,B,C,D,E
B) A,B,C
C) C,D,E
D) A,B,D,E

答案：A
解析：自然连接保留所有属性，公共属性C只出现一次。
```

**例题2：范式判断**
```
关系模式R(A,B,C,D)，函数依赖集F={A→B, B→C, C→D, D→A}，
R的候选键是？
A) A
B) B
C) C
D) A,B,C,D都是

答案：D
解析：由于存在循环依赖，每个属性都可以确定其他所有属性。
```

## 软件工程
**重要程度：高 | 考查频度：很高**

### 核心知识点

#### 1. 软件生命周期与模型
- **瀑布模型**：线性顺序，适合需求明确的项目
- **增量模型**：分批交付，逐步完善
- **原型模型**：快速原型，用户反馈
- **螺旋模型**：风险驱动，适合大型复杂项目
- **敏捷方法**：
  - Scrum：迭代开发，Sprint周期
  - XP（极限编程）：结对编程，测试驱动

#### 2. 软件开发方法
- **结构化方法**：
  - 数据流图（DFD）
  - 实体关系图（ERD）
  - 状态转换图（STD）
- **面向对象方法**：
  - 用例驱动
  - 架构为中心
  - 迭代增量

#### 3. 软件测试
- **测试分类**：
  - 按阶段：单元测试、集成测试、系统测试、验收测试
  - 按方法：黑盒测试、白盒测试、灰盒测试
- **用例设计**：
  - 等价类划分
  - 边界值分析
  - 判定表驱动
- **McCabe复杂度**：环路复杂度V(G) = E - N + 2

#### 4. 质量与维护
- **软件质量特性**：功能性、可靠性、易用性、效率、维护性、可移植性
- **维护类型**：
  - 纠错性维护：修复错误
  - 适应性维护：环境变化
  - 完善性维护：功能增强
  - 预防性维护：提高可维护性
- **CMMI过程改进**：5个成熟度等级

### 典型例题

**例题1：软件生命周期模型**
```
下列哪种软件开发模型最适合需求不明确、用户参与度高的项目？
A) 瀑布模型
B) 螺旋模型
C) 原型模型
D) V模型

答案：C
解析：原型模型通过快速构建原型获得用户反馈，适合需求不明确的情况。
```

**例题2：软件测试**
```
对于输入域为[1,100]的程序，采用边界值分析法，应该选择的测试用例是？
A) 1, 50, 100
B) 0, 1, 100, 101
C) 1, 2, 99, 100
D) 0, 1, 2, 99, 100, 101

答案：D
解析：边界值分析包括边界值及其邻近值。
```

## 面向对象技术
**重要程度：高 | 考查频度：高**

### 核心知识点

#### 1. 核心概念
- **类与对象**：
  - 类：对象的模板
  - 对象：类的实例
- **封装**：数据和方法的结合，信息隐藏
- **继承**：子类继承父类的属性和方法
- **多态**：同一接口的不同实现
- **消息传递**：对象间的通信机制

#### 2. UML建模
- **用例图**：系统功能和用户交互
- **类图**：类的结构和关系
- **时序图**：对象间的时间顺序交互
- **活动图**：业务流程和算法流程
- **状态图**：对象状态变化
- **协作图**：对象间的协作关系

#### 3. 设计模式
- **创建型模式**：
  - 单例模式：确保类只有一个实例
  - 工厂模式：创建对象的接口
- **结构型模式**：
  - 适配器模式：接口转换
  - 装饰器模式：动态添加功能
- **行为型模式**：
  - 观察者模式：一对多依赖关系
  - 策略模式：算法族的封装

### 典型例题

**例题1：面向对象概念**
```
下列关于多态的描述，正确的是？
A) 多态是指一个类可以有多个父类
B) 多态是指同一个方法名可以有不同的实现
C) 多态是指一个对象可以属于多个类
D) 多态是指一个类可以有多个构造函数

答案：B
解析：多态允许同一接口有不同的实现，运行时确定具体调用哪个实现。
```

**例题2：UML类图**
```
在UML类图中，表示"聚合"关系的符号是？
A) 实线箭头
B) 虚线箭头
C) 空心菱形
D) 实心菱形

答案：C
解析：聚合关系用空心菱形表示，组合关系用实心菱形表示。
```

## 网络与安全
**重要程度：中 | 考查频度：中等**

### 核心知识点

#### 1. 网络基础
- **OSI七层模型**：
  - 物理层、数据链路层、网络层、传输层
  - 会话层、表示层、应用层
- **TCP/IP四层模型**：
  - 网络接口层、网际层、传输层、应用层
- **IP地址与子网划分**：
  - IPv4地址分类：A、B、C类
  - 子网掩码、CIDR表示法
- **网络设备**：
  - 集线器（Hub）、交换机（Switch）
  - 路由器（Router）、网关（Gateway）

#### 2. 网络安全
- **加密技术**：
  - 对称加密：DES、AES
  - 非对称加密：RSA、ECC
- **数字签名**：身份认证和数据完整性
- **防火墙**：包过滤、状态检测、应用层网关
- **常见攻击**：
  - DoS/DDoS攻击
  - SQL注入、XSS攻击
  - 中间人攻击

#### 3. 信息安全法规
- **网络安全法**：基本原则和要求
- **个人信息保护法**：数据处理规范
- **数据安全法**：数据分类分级保护

### 典型例题

**例题1：网络协议**
```
TCP协议属于OSI模型的哪一层？
A) 网络层
B) 传输层
C) 会话层
D) 应用层

答案：B
解析：TCP是传输层协议，负责可靠的端到端通信。
```

**例题2：IP地址计算**
```
IP地址192.168.1.100/26的网络地址是？
A) 192.168.1.0
B) 192.168.1.64
C) 192.168.1.96
D) 192.168.1.128

答案：B
解析：/26表示前26位是网络位，子网掩码为255.255.255.192，
网络地址为192.168.1.64。
```

## 计算机组成原理
**重要程度：中 | 考查频度：中等**

### 核心知识点

#### 1. 数据表示与运算
- **数制转换**：二进制、八进制、十六进制
- **补码表示**：正数、负数的二进制表示
- **浮点数**：IEEE 754标准，符号位、指数位、尾数位
- **算术运算**：加法、减法、乘法、除法
- **逻辑运算**：与、或、非、异或

#### 2. CPU组成
- **ALU（算术逻辑单元）**：执行算术和逻辑运算
- **控制器**：指令解码、控制信号生成
- **寄存器**：程序计数器、指令寄存器、累加器
- **指令系统**：
  - CISC：复杂指令集
  - RISC：精简指令集
- **流水线技术**：指令重叠执行，提高效率

#### 3. 存储系统
- **存储层次结构**：
  - 寄存器 → Cache → 内存 → 外存
- **Cache存储器**：
  - 直接映射、全相联、组相联
  - 命中率、访问时间
- **磁盘存储**：
  - RAID技术：RAID 0、1、5、10
  - 磁盘调度算法

### 典型例题

**例题1：数据表示**
```
十进制数-25的8位补码表示是？
A) 11100111
B) 10011001
C) 11100110
D) 10011000

答案：A
解析：25的二进制为00011001，取反得11100110，加1得11100111。
```

**例题2：Cache存储**
```
某计算机Cache容量为64KB，块大小为64B，采用直接映射方式。
主存地址为32位，则Cache的标记位数是？
A) 20
B) 21
C) 22
D) 23

答案：C
解析：块内地址6位，Cache索引10位，标记位=32-6-10=16位。
实际应为22位，需要重新计算。
```

# 信息安全加密与认证技术笔记

本节整理了常见加密技术、数字签名、信息摘要、数字证书及身份认证的知识点和学习口诀，便于理解与复习。

---

## 一、加密技术分类

### 1. 对称加密（Symmetric Encryption）
- **定义**：加密和解密使用同一密钥。
- **特点**：
  - 密钥只能双方拥有，需要安全渠道传递
  - 加密深度较低，效率高，适合大数据量
- **常见算法**：
  - DES、3DES、AES、RC5、IDEA
- **口诀**：3S结尾 + 五一（DES、3DES、AES、RC5、IDEA）
- **分类**：
  - 分组加密：DES、3DES、AES、IDEA
  - 流密码：RC5
- **示意流程**：
    发送者A → 使用密钥加密 → 互联网上传 → 接收者B → 使用密钥解密 → 明文

---

### 2. 非对称加密（Asymmetric Encryption / Public Key Encryption）
- **定义**：加密和解密使用不同的密钥，公钥可公开，私钥仅自己拥有。
- **特点**：
  - 不需专门通道传递密钥
  - 加密深度高，效率低，适合小数据量
- **常见算法**：
  - RSA、DSA、ECC
- **口诀**：2A2C（RSA、DSA、ECC）
- **示意流程**：
    1. 接收者B生成公钥PB与私钥SB
    2. 发送者A用PB加密消息
    3. 互联网上传密文
    4. B用SB解密 → 明文

---

## 二、数字签名与信息摘要

### 1. 信息摘要（Hash）
- **作用**：实现数据完整性
- **算法**：
  - MD5 → 128位
  - SHA → 160位
- **特点**：
  - 将任意长度的消息压缩为固定长度
  - 可检测信息是否被篡改

### 2. 数字签名（Digital Signature）
- **作用**：实现认证和不可抵赖性
- **流程**：
    发送者A → 消息摘要 → 用A私钥加密 → 生成签名 → 互联网上传 → 接收者B用A公钥解密 → 验证发送者身份  
- **效果**：
  - 确认发送者真实性
  - 接收者无法伪造或篡改消息
  - 与非对称加密不同：使用A自己的私钥加密，非对称加密用于数据保密时用接收者公钥

### 3. 典型题目关联
| 问题 | 解决方案 |
|------|----------|
| 数字签名用途 | 认证，确保发送者身份 |
| 信息完整性 | 使用摘要算法（MD5/SHA） |
| 对大量数据加密 | 对称加密，效率高 |
| 认证/加密区别 | 加密保障保密性，签名保障真实性/不可抵赖性 |

## 三、身份认证与数字证书

### 1. 身份认证（Authentication）
- **作用**：确认实体是否合法
- **方式**：
  - 数字证书（由CA签发）
  - 确认公钥和证书签名
- **流程**：
    1. 服务端提交证书至CA审核
    2. CA签发证书（内含公钥、服务端身份信息及CA签名）
    3. 客户端下载证书，验证CA签名 → 确认服务合法性

### 2. 安全通信结合对称/非对称加密
- **流程**：
    1. 客户端下载服务端证书 → 验证合法性
    2. 生成随机对称密钥 → 用服务端公钥加密传送
    3. 服务端解密获取对称密钥
    4. 双方用对称密钥加密数据 → 高效安全通信

### 3. 互信条件
- AB双方要互相信任：
  - 需要获取对方的公钥
  - 通过数字证书验证合法性

## 四、综合对比

| 技术 | 用途 | 公钥/私钥 | 优点 | 缺点 | 适用场景 |
|------|------|------------|------|------|----------|
| 对称加密 | 数据加密 | 同一密钥 | 高效，适合大数据 | 密钥分发困难 | 文件传输、大数据加密 |
| 非对称加密 | 数据加密 | 公钥加密/私钥解密 | 不需专门通道 | 速度慢 | 小数据量、密钥交换 |
| 信息摘要 | 完整性 | 无密钥 | 检测篡改 | 不能加密 | 完整性验证 |
| 数字签名 | 认证/不可抵赖 | 私钥签名/公钥验证 | 认证明确，不可否认 | 速度慢 | 认证消息、电子合同 |
| 数字证书 | 身份认证 | 公钥/CA签名 | 确认合法性 | 依赖CA | HTTPS、安全通信 |

## 五、口诀回顾

- **对称加密**：3S结尾 + 五一（DES、3DES、AES、RC5、IDEA）
- **非对称加密**：2A2C（RSA、DSA、ECC）
- **摘要算法**：
  - MD5 → 128位 → 较弱
  - SHA → 160位 → 较强

## 六、学习提示

1. 对称与非对称加密的公私钥使用场景不同：
   - 数据加密 → 用对方公钥加密，接收方私钥解密
   - 数字签名 → 用自己私钥加密摘要，接收方用自己公钥解密验证
2. 身份认证与认证/不可抵赖性区别：
   - 身份认证 → 证书验证合法性
   - 数字签名 → 验证消息来源真实性
3. 常用安全组合：
   - 数字证书 + 非对称加密 → 安全交换对称密钥
   - 对称加密 → 高效数据传输
   - 信息摘要 + 数字签名 → 保障完整性与认证
